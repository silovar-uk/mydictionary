# 設計判断

## 登録を最優先

見出し語のみで保存できる。意味・出典・タグを必須にしない。登録後に詳細編集で育てる。

## IndexedDBを正本にする

Dexieを介してIndexedDBへ保存する。検索用データは再構築可能な副データとし、復元元にしない。

## 「ランダム」ではなく「開く」

通常モードは完全乱数ではなく、未閲覧・長期間未閲覧・未表示・育て途中を少し優先する。完全ランダムも別モードで残す。

## データを閉じ込めない

JSON完全バックアップ、CSV、TXTをMVPに含める。サーバー・ログイン・外部分析は導入しない。

## 紙らしさは組版でつくる

明朝体、細い罫線、頁番号、索引的なメタ情報で辞書らしさをつくる。強いセピアや3Dページめくりは使わない。
